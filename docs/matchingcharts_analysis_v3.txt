PART I: CONCEPTUAL FRAMEWORK
================================================================================

i) Difference between "Optimization Variables (Solved)" and "Outputs (Results)"

This is an important conceptual distinction:

OPTIMIZATION VARIABLES (Solved) are the unknowns that the solver (fsolve)
iteratively adjusts to satisfy the system of constraint equations. These are
the variables we seek as part of the optimization problem.

In the MATLAB script (lines 8-12), these are defined as optimvar() objects
that the solver manipulates:

  Variable    Symbol      Description
  ────────────────────────────────────────────────────────────
  W           W           Total weight (MTOW)
  S_W         W/S         Wing loading (weight/wing area)
  T_S         T/S         Thrust loading (thrust/wing area)
  lam         λ           Aspect ratio (bounded [6, 12])
  k           k           Fuel fraction (W_fuel/W)

OUTPUTS (Results) are derived quantities calculated from the solved
optimization variables AFTER convergence. These are not unknowns—they are
post-processing computations:

  Output      Formula             Description
  ────────────────────────────────────────────────────────────
  S           S = W / (W/S)       Wing area
  b           b = √(S × λ)        Wingspan
  c           (MAC formula)       Mean aerodynamic chord
  T           T = (T/S) × S       Total thrust
  Cl, Cd, L/D (aerodynamic calcs) Cruise aerodynamics

The distinction is:
- Optimization variables: Inputs to the solver, iterated to find solution
- Outputs: Computed from the solution, reported for downstream design

--------------------------------------------------------------------------------

ii) Reynolds Number: Calculation, Output, or Hypothesis?

Examining the MATLAB script (lines 69, 175):

  % Line 69 (before solver):
  Rewing = V*c/nu;

  % Line 175 (after solver):
  Rewing = V*c/nu;

The Reynolds number is an OUTPUT, computed from other parameters and the
solved MAC (c). The calculation chain is:

  1. V (cruise velocity) = 221.2 m/s → INPUT (hypothesis/fixed constant)
  2. nu (kinematic viscosity) = 2.44×10⁻⁵ m²/s → INPUT (atmosphere property)
  3. c (MAC) → OUTPUT (computed from solved S, λ, and taper ratio)
  4. Re = V × c / nu → OUTPUT (computed from 1, 2, 3)

So, for the regional jet:
- Re is neither a hypothesis nor an input
- Re is a derived output from wing geometry that emerges from the sizing

This is important for airfoil selection: The sizing script must be run FIRST
to obtain the wing geometry (S, b, c), and THEN the Reynolds number can be
calculated to feed into XFOIL or experimental data lookup.

H. LOAD FACTORS AND STRUCTURAL DESIGN (NEW SECTION)
--------------------------------------------------------------------------------

The original regional jet script used load factors from CS-25 (large aircraft):

  Nult = 1.5 × 3.8 = 5.7  (ultimate load factor)

This value was used in structural weight estimation formulas:
  - Qala (wing mass): Sadraey Eq. 10.3, includes (λ × Nult)^0.6 term
  - Qfus (fuselage mass): includes Nult × (W/g) × Lfus term

For the Mars UAV, CS-25 is NOT applicable. The appropriate reference is:

  EASA CS-23 Amendment 5 (2017): Certification Specifications for
  Normal-Category Aeroplanes
  
  Reference: [@europeanunionaviationsafetyagencyCertificationSpecificationsNormalCategory2017]

However, CS-23 applies to manned aircraft up to 8,618 kg MTOM. For an
unmanned Mars aircraft, there is NO regulatory certification requirement.
The load factor should be selected based on ENGINEERING JUDGMENT considering:

1. Mission profile (expected maneuvers, gust encounters)
2. Absence of crew (lower safety requirement than manned aircraft)
3. Mars atmospheric conditions (thin atmosphere → smaller gust loads)

PROPOSED LOAD FACTOR FOR MARS UAV:

Parameter               Symbol          Mars UAV           Regional Jet (CS-25)
────────────────────────────────────────────────────────────────────────────────
Positive limit load     n_limit+        +2.5               +3.8
Negative limit load     n_limit-        -1.0               -1.52
Factor of safety        SF              1.5                1.5
Ultimate load factor    n_ult           3.75               5.7

Rationale for n_limit = +2.5:
- No pilot → no injury risk from high-g maneuvers
- Autonomous flight → predictable, limited maneuvers
- Mars atmosphere 60× thinner → gust loads reduced proportionally
- Value consistent with lightweight UAV practice

Rationale for keeping SF = 1.5:
- Universal engineering practice for aerospace structures
- Accounts for material variability and manufacturing tolerances
- Provides margin for unforeseen loading conditions

STRUCTURAL WEIGHT ESTIMATION NOTE:
The structural weight estimation formulas (Sadraey, Raymer, etc.) must be
recalculated using the Mars-appropriate n_ultimate = 3.75 instead of the
CS-25 value of 5.7. This will reduce estimated structural mass.

OPTIMIZATION VARIABLES (Unknowns to be Solved)
================================================================================

These are the variables that the sizing algorithm solves for:

Variable    Mars UAV              Regional Jet           Difference
────────────────────────────────────────────────────────────────────────────────
W           Total weight (N)      Total weight (N)       ✅ Same concept
W/S         Wing loading (N/m²)   Wing loading (N/m²)    ✅ Same concept
λ           Aspect ratio          Aspect ratio           ✅ Same concept
P/W         Power loading (W/N)   T/S (thrust loading)   ⚠️ CHANGED to power
            or (W/kg)

Notes:
- T/S → P/W: For electric propulsion, POWER loading is more natural than
  THRUST loading because we size by power consumption, not thrust production.
  
  Relationship: P = T × V / η_prop (for cruise)
                P = T × V_induced / FM (for hover, from momentum theory)

- Units for P/W:
  * W/N (watts per newton of weight) — dimensionally consistent
  * W/kg (watts per kilogram) — more intuitive for electric aircraft

- Battery fraction f_batt is NOT an optimization variable for electric aircraft
  because battery mass is constant. Instead, battery energy E_batt is an OUTPUT
  that must satisfy the mission energy requirement.

PART III: CONSTRAINT EQUATIONS FOR MARS UAV
================================================================================

The Mars UAV requires 5 constraint equations (same count as regional jet):

#   Mars UAV Constraint                              Replaces (Regional Jet)
────────────────────────────────────────────────────────────────────────────────
1   Mass balance                                     Mass balance (same structure)
    Σ(component masses) = MTOW                       Σ(Q terms) × g = W

2   Hover power constraint                           ❌ Take-off runway
    P_hover ≥ f(W, ρ, A_rotor, FM)                  (momentum theory)

3   Stall constraint                                 ❌ Landing runway
    W/S ≤ ½ρV_min²C_L,max                           (minimum safe speed)

4   Cruise power constraint                          Cruise thrust (modified)
    P_cruise = D × V / η = (W/L/D) × V / η          P = T × V / η

5   Energy constraint                                ❌ Breguet range
    E_batt ≥ P_hover×t_hover + P_cruise×t_cruise    (battery must store
    + E_reserve                                      mission energy)

--------------------------------------------------------------------------------
Constraint Equation Details
--------------------------------------------------------------------------------

1. MASS BALANCE (same concept as regional jet)

   MTOW = m_struct + m_prop + m_batt + m_payload + m_avionics + m_thermal

   Or using fractions:
   1 = f_struct + f_prop + f_batt + f_payload + f_avionics + f_thermal
   
   NOTE: Structural mass (m_struct) is calculated using weight estimation
   formulas that include n_ultimate. Use n_ult = 3.75 for Mars UAV.

2. HOVER POWER CONSTRAINT (replaces takeoff runway)

   From momentum theory (Leishman):
   
   P_hover = T^(3/2) / (FM × √(2ρA))
   
   Where:
   - T = W (thrust = weight in hover)
   - FM = figure of merit (0.6-0.7)
   - A = total rotor disk area
   - ρ = atmospheric density

   For n rotors of diameter D:
   A = n × π × D² / 4

3. STALL CONSTRAINT (replaces landing runway)

   The wing must generate enough lift at minimum speed:
   
   W/S ≤ ½ × ρ × V_min² × C_L,max
   
   This sets the MAXIMUM allowable wing loading.

4. CRUISE POWER CONSTRAINT (modified from regional jet)

   Power required to overcome drag at cruise:
   
   P_cruise = D × V / η_total = (W / (L/D)) × V / η_total
   
   Where:
   - L/D = lift-to-drag ratio at cruise C_L
   - η_total = η_prop × η_motor × η_ESC

5. ENERGY CONSTRAINT (replaces Breguet range)

   Total mission energy requirement:
   
   E_mission = P_hover × t_hover + P_cruise × t_cruise
   
   Battery must provide:
   
   E_batt = E_mission / (DoD × (1 - reserve))
   
   Battery mass:
   
   m_batt = E_batt / e_spec

--------------------------------------------------------------------------------
NOTE ON TRANSITION (Post-Sizing Verification, NOT a Constraint)
--------------------------------------------------------------------------------

For hybrid VTOL aircraft (QuadPlane configuration), transition from hover to
forward flight requires the wing to progressively take over lift from the
rotors. This is NOT included as a sizing constraint because:

1. Literature review shows that proper transition corridor analysis requires
   full 6-DOF dynamic modeling with equilibrium equations involving:
   - Wing lift: L = ½ρV²SC_L
   - Rotor thrust components (vertical and horizontal)
   - Pitch angle and angle of attack constraints
   - Available power limits
   
   Reference: Zhao et al. (2023), "Development of Multimode Flight Transition
   Strategy for Tilt-Rotor VTOL UAVs", Drones 7(9):580, DOI:10.3390/drones7090580

2. No simple "wing lift fraction α" parameter is defined in the literature
   for preliminary sizing. The α ≈ 0.5-0.7 value previously proposed was
   not properly sourced and has been removed.

3. The stall constraint (W/S ≤ ½ρV_min²C_L,max) already ensures that the
   wing CAN generate full lift at the minimum speed V_min.

4. The hover power constraint ensures rotors CAN support full weight in hover.

5. By continuity, if both constraints are satisfied, a feasible transition
   path exists where lift is progressively transferred from rotors to wing.

POST-SIZING VERIFICATION:
After sizing, verify that the transition speed satisfies:

   V_transition ≥ 1.2 × V_stall

Where V_stall = V_min from the stall constraint. The factor 1.2 is standard
aviation practice (similar to 1.2 Vs for takeoff safety speed in conventional
aircraft). This provides margin for gusts and control authority during the
transition phase.

If detailed transition corridor analysis is required (for final design), use
the full equilibrium equations from Zhao et al. (2023) or equivalent methods.
