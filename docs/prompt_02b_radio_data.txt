CONTEXT
================================================================================

You are working on a Mars UAV feasibility study. The UAV has a TELECOMMUNICATION
RELAY mission objective to extend communication range for surface operations.

This task collects reference data on existing radio systems to:
- Understand what types of radios are suitable for UAV relay missions
- Establish realistic mass ranges for radio payloads
- Provide grounded data for initial MTOW estimation

The YAML output will be used by another LLM agent to:
- Write the manuscript content for Section 3.1 (Payload systems survey)
- Justify payload mass allocation in Section 3.4 (Initial weight estimate)
- Inform component selection in Section 11.4 (after preliminary sizing)

NOTE: For Mars, the radio would communicate with the habitat ground station
or EVA astronauts, not Earth. The frequency bands differ from Earth applications
but mass references remain valid for estimation purposes.

================================================================================
DATA FIELDS TO COLLECT
================================================================================

For each radio system, collect ALL of the following fields.
Use null for any field where data is not available from the primary source.

| Field                | Description                              | Unit    |
|----------------------|------------------------------------------|---------|
| model                | Full product model name                  | string  |
| manufacturer         | Company name                             | string  |
| type                 | "mesh" or "point-to-point" or "c2_link"  | string  |
| mass_radio_g         | Radio/transceiver unit mass only         | grams   |
| mass_antenna_g       | Standard antenna mass                    | grams   |
| mass_total_g         | Complete system as configured for UAV    | grams   |
| frequency_band       | e.g., "2.4 GHz", "900 MHz", "L-band"     | string  |
| frequency_range_mhz  | e.g., "2400-2500"                        | string  |
| channel_bandwidth_mhz| e.g., "1.25, 2.5, 5, 10, 20"             | string  |
| output_power_dbm     | Maximum transmit power                   | number  |
| output_power_w       | Maximum transmit power in watts          | number  |
| data_rate_mbps       | Maximum data rate                        | number  |
| data_rate_typical_mbps| Typical/sustained data rate             | number  |
| range_los_km         | Line-of-sight range                      | number  |
| range_nlos_km        | Non-line-of-sight range (if applicable)  | number  |
| latency_ms           | Typical latency                          | number  |
| voltage_input_v      | Operating voltage range                  | string  |
| power_consumption_w  | Typical power draw                       | number  |
| power_consumption_max_w | Maximum power draw (at max TX)         | number  |
| operating_temp_min_c | Minimum operating temperature            | number  |
| operating_temp_max_c | Maximum operating temperature            | number  |
| dimensions_mm        | "L x W x H"                              | string  |
| ip_rating            | e.g., "IP67"                             | string  |
| encryption           | e.g., "AES-256"                          | string  |
| mimo_config          | e.g., "2x2", "4x4"                       | string  |
| interface            | e.g., "Ethernet", "USB", "UART"          | string  |
| antenna_connectors   | e.g., "2x SMA", "4x TNC"                 | string  |
| certifications       | e.g., "FCC, CE, MIL-STD-810G"            | string  |
| source_url           | Primary manufacturer URL                 | string  |
| source_key           | Zotero citation key                      | string  |
| access_date          | Date accessed (YYYY-MM-DD)               | string  |
| notes                | Any relevant notes                       | string  |
